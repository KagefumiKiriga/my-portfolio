# 2026/07/20 開発日記：継続抽出ノードの変数参照バグ調査＆解決

## 今日やったこと

CrowdWorks案件判定ワークフローの「HTTP リクエスト2」ノードで、
「継続抽出」ノードのcontinuous変数が変数選択メニューに出てこない不具合を調査・解決した。

## 起きていた問題

- 「継続抽出」ノード自体は正常（コード・出力変数設定ともに問題なし）
- 配線も正しく繋がっていた（判定LLM→継続抽出→レベル抽出→…）
- にもかかわらず、HTTPリクエスト2の変数選択メニューにcontinuousが候補に出てこない

## 調査した候補と結果

1. JSON構文エラー（`"continuous": "{"`のカギカッコ崩れ）→ 修正したが解決せず
2. ノードの出力変数の作り直し → 解決せず
3. ブラウザの完全リロード → 解決せず
4. HTTP リクエスト2が別ノードである可能性を確認 → 正しいノードだった
5. **配線の切断→再接続 → これで解決！**

## 原因（推測）

Difyの内部依存関係キャッシュが、リネームやノード編集後に更新されず古いままになっていた。
配線を一度切って繋ぎ直すことで、依存関係が再計算され、正しく変数が見えるようになった。

## 学び

- ノード自体・配線が正常に見えても、変数が候補に出ない場合は「内部キャッシュの不整合」を疑う
- 対処法：該当の配線を一度削除して繋ぎ直す（force-refresh的な効果がある）
- それでもダメな場合はノードを複製して繋ぎ替える方法もある（内部IDが完全に新しくなる）
- 0717の「ノードのリネームで参照が壊れる」問題と根っこは同じ系統（Difyの参照キャッシュ系の不具合）
